# Decisions

## ADR-001. Использовать Astro для сайта-блога

Дата: 2026-06-29

Статус: принято

Решение: использовать Astro как основной фреймворк.

Причина:

- проект является контентным сайтом;
- Astro поддерживает Markdown и MDX;
- Astro подходит для статической генерации;
- проект должен запускаться локально без серверной инфраструктуры.

## ADR-002. Хранить статьи локально в Markdown/MDX

Дата: 2026-06-29

Статус: принято

Решение: хранить статьи в `src/content/articles/` в формате Markdown или MDX.

Причина:

- текущий этап не включает базу данных и админ-панель;
- контент должен быть простым для версионирования в Git;
- Astro имеет встроенную поддержку контентных коллекций.

## ADR-003. Не использовать базу данных на текущем этапе

Дата: 2026-06-29

Статус: принято

Решение: не добавлять базу данных в MVP.

Причина:

- ТЗ описывает локальный сайт с хранением контента в файлах;
- база данных усложнит локальную разработку;
- текущие требования покрываются статическими файлами и сборкой Astro.

## ADR-004. Использовать Tailwind CSS и CSS-переменные

Дата: 2026-06-29

Статус: принято

Решение: использовать Tailwind CSS для стилей и CSS-переменные для цветовых тем.

Причина:

- Tailwind CSS указан в рекомендуемом стеке;
- CSS-переменные позволяют централизованно менять цвета темы;
- class-based theme switching соответствует требованиям ТЗ.

## ADR-005. Хранить UI-состояние в localStorage

Дата: 2026-06-29

Статус: принято

Решение: сохранять выбранную тему и состояние сайдбара в `localStorage`.

Причина:

- ТЗ прямо указывает `localStorage` для этих сценариев;
- состояние относится только к интерфейсу пользователя;
- серверное хранение на текущем этапе не требуется.

## ADR-006. Не добавлять интеграции будущей экосистемы в MVP

Дата: 2026-06-29

Статус: принято

Решение: не добавлять RAG, ИИ-чат, генерацию видео, публикацию в соцсети, авторизацию, платежи и внешние LLM на текущем этапе.

Причина:

- эти функции явно исключены из текущего этапа;
- сайт должен оставаться независимым проектом;
- будущие интеграции должны проектироваться отдельно.
