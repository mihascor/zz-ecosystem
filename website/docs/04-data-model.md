# Data Model

## Источник данных

На текущем этапе база данных не используется.

Контент хранится локально в Markdown или MDX-файлах внутри проекта.

## Article

Сущность статьи задаётся frontmatter в файле Markdown/MDX.

Пример:

```md
---
title: "Введение в Запретные знания"
slug: "vvedenie-v-zapretnye-znaniya"
order: 1
category: "Основы"
subcategory: "Введение"
tags:
  - сознание
  - знание
  - путь
cover: "/images/articles/vvedenie.jpg"
description: "Вводная статья проекта «Запретные знания»."
published: true
date: "2026-06-29"
---

Текст статьи...
```

### Поля статьи

| Поле | Тип | Обязательное | Назначение |
| --- | --- | --- | --- |
| `title` | string | да | Название статьи |
| `slug` | string | да | Человекочитаемый URL статьи |
| `order` | number | да | Порядок для последовательного чтения |
| `category` | string | да | Основная категория |
| `subcategory` | string | да | Подкатегория |
| `tags` | string[] | да | Список тегов |
| `cover` | string | да | Путь к изображению статьи |
| `description` | string | да | Краткое описание |
| `published` | boolean | да | Показывать или скрыть статью |
| `date` | string | да | Дата публикации |

## Category

Категория не хранится отдельной таблицей или файлом на текущем этапе.

Категории вычисляются из поля `category` опубликованных статей.

## Subcategory

Подкатегория не хранится отдельной таблицей или файлом на текущем этапе.

Подкатегории вычисляются из поля `subcategory` опубликованных статей и используются внутри соответствующих категорий.

## Tag

Тег не хранится отдельной таблицей или файлом на текущем этапе.

Теги вычисляются из поля `tags` опубликованных статей.

## Связи

- Одна статья относится к одной категории.
- Одна статья относится к одной подкатегории.
- Одна статья может иметь несколько тегов.
- Одна категория содержит несколько подкатегорий.
- Одна подкатегория содержит несколько статей.
- Один тег может быть связан с несколькими статьями.

## Изображения

Изображения статей можно хранить в:

```text
public/images/articles/
```

Путь к изображению указывается в поле `cover`, например:

```text
/images/articles/vvedenie.jpg
```

Файлы изображений должны иметь понятные имена и быть оптимизированы по размеру.
