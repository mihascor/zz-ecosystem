# Requirements

## Функциональные требования

### Контент

- Статьи хранятся локально в Markdown или MDX.
- Каждая статья должна иметь frontmatter с обязательными полями:
  - `title`;
  - `slug`;
  - `order`;
  - `category`;
  - `subcategory`;
  - `tags`;
  - `cover`;
  - `description`;
  - `published`;
  - `date`.
- Опубликованные статьи отображаются на сайте.
- Статьи с `published: false` не отображаются в пользовательских списках и навигации.
- Изображения статей можно хранить в `public/images/articles/`.

### Главная страница `/`

На главной странице должны быть:

- название проекта;
- краткое описание проекта;
- список последних или рекомендуемых статей;
- переход ко всем статьям;
- доступ к категориям;
- доступ к тегам.

### Страница всех статей `/articles`

Страница должна:

- выводить все опубликованные статьи;
- поддерживать сортировку по `order` или дате;
- показывать название, описание, категорию, теги и обложку;
- вести на страницу конкретной статьи.

### Страница статьи `/articles/[slug]`

На странице статьи должны быть:

- заголовок;
- изображение;
- дата публикации;
- категория;
- подкатегория;
- теги;
- основной текст;
- навигация «предыдущая статья / следующая статья»;
- хлебные крошки;
- левый сайдбар с деревом статей;
- кнопка скрытия сайдбара;
- переключатель темы.

### Страница категории `/categories/[category]`

Страница должна:

- выводить все статьи выбранной категории;
- отображать подкатегории;
- вести в статьи;
- показывать теги, относящиеся к категории.

### Страница подкатегории `/subcategories/[subcategory]`

Страница должна:

- выводить все статьи выбранной подкатегории;
- сортировать статьи по `order`;
- вести в конкретную статью.

### Страница тега `/tags/[tag]`

Страница должна:

- выводить все статьи с выбранным тегом;
- показывать количество статей;
- вести в конкретную статью.

## Сайдбар

Сайдбар должен автоматически строиться на основе статей и показывать:

- категории;
- подкатегории;
- статьи внутри подкатегорий.

Сайдбар должен:

- подсвечивать текущую статью;
- позволять переходить между статьями;
- скрываться и открываться обратно;
- запоминать состояние в `localStorage`;
- корректно работать на мобильных устройствах;
- иметь действие «развернуть все» и «скрыть все» для категорий, подкатегорий и статей.

## Темы

Сайт должен поддерживать:

- светлую тему;
- тёмную тему;
- системную тему.

Поведение темы:

- пользователь может переключить тему вручную;
- выбранная тема сохраняется в `localStorage`;
- при первом открытии используется сохранённая тема, затем системная тема, затем светлая тема по умолчанию;
- оформление должно меняться для фона, текста, сайдбара, карточек, ссылок, кнопок, блоков кода, цитат, хлебных крошек и навигации по статьям.

## Интерфейс

Интерфейс должен быть минималистичным, читаемым и удобным для длительного чтения.

Обязательные элементы:

- шапка;
- левый сайдбар;
- основная область контента;
- переключатель темы;
- кнопка скрытия сайдбара;
- хлебные крошки;
- список тегов;
- навигация по статьям.

Сайт должен корректно отображаться на десктопе, планшете и мобильном устройстве. На мобильных устройствах сайдбар должен быть скрываемым и выезжающим.

## SEO

На текущем этапе сайт должен поддерживать:

- уникальный `title` для каждой страницы;
- `description` для каждой статьи;
- человекочитаемые URL;
- Open Graph-разметку;
- favicon;
- `robots.txt`;
- возможность генерации карты сайта, если она понадобится для будущего размещения.

## Критерии приёмки

Проект считается выполненным, если:

1. Сайт запускается локально.
2. Главная страница открывается без ошибок.
3. Есть страница списка статей.
4. Есть минимум несколько тестовых статей.
5. Каждая статья открывается по отдельному URL.
6. У статьи отображаются заголовок, изображение, описание, категория, подкатегория и теги.
7. Работает навигация «предыдущая / следующая статья».
8. Работает левый сайдбар.
9. Сайдбар можно скрыть и вернуть обратно.
10. Состояние сайдбара сохраняется после перезагрузки.
11. Работают страницы категорий.
12. Работают страницы подкатегорий.
13. Работают страницы тегов.
14. Работает переключение светлой и тёмной темы.
15. Выбранная тема сохраняется после перезагрузки.
16. Сайт корректно отображается на десктопе и мобильном устройстве.
17. Проект успешно собирается командой build.
18. В проекте есть README с инструкцией запуска.
