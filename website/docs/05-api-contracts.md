# API Contracts

## Текущий этап

На текущем этапе внешние API и серверные контракты не используются.

Сайт работает как локальный Astro-проект со статическими страницами и локальными Markdown/MDX-файлами.

## Внутренние контракты данных

Вместо HTTP API используются внутренние функции чтения и преобразования контента.

### Получение опубликованных статей

Назначение:

- получить статьи из `src/content/articles/`;
- исключить статьи с `published: false`;
- вернуть данные для списков, сайдбара, страниц категорий, подкатегорий и тегов.

Ожидаемая форма результата:

```ts
type Article = {
  title: string;
  slug: string;
  order: number;
  category: string;
  subcategory: string;
  tags: string[];
  cover: string;
  description: string;
  published: boolean;
  date: string;
};
```

### Навигация по статьям

Назначение:

- отсортировать опубликованные статьи по `order`;
- определить предыдущую и следующую статью для текущей статьи.

Ожидаемая форма результата:

```ts
type ArticleNavigation = {
  previous: Article | null;
  next: Article | null;
};
```

### Дерево категорий

Назначение:

- построить структуру сайдбара на основе опубликованных статей;
- сгруппировать статьи по категориям и подкатегориям;
- отсортировать статьи внутри подкатегорий по `order`.

Ожидаемая форма результата:

```ts
type CategoryTree = {
  category: string;
  subcategories: {
    subcategory: string;
    articles: Article[];
  }[];
}[];
```

### Теги

Назначение:

- получить список тегов из опубликованных статей;
- связать каждый тег со статьями;
- показать количество статей для страницы тега.

Ожидаемая форма результата:

```ts
type TagIndex = {
  tag: string;
  count: number;
  articles: Article[];
}[];
```

## Будущие интеграции

Возможные будущие связи с другими проектами экосистемы должны проектироваться отдельно. На текущем этапе не нужно добавлять контракты для RAG-системы, ИИ-чата, генерации видео, публикации в соцсети, авторизации или платежей.
