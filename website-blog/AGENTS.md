# AGENTS.md

## Контекст проекта

Этот репозиторий содержит проект сайта.

Агент должен вносить небольшие, безопасные и понятные изменения, избегая ненужного рефакторинга.

## Общие правила

- Перед изменением файлов изучи существующую структуру проекта.
- Сохраняй текущую архитектуру, стиль именования, форматирование и структуру папок.
- Не переименовывай файлы, не перемещай папки и не меняй публичные API, если задача явно этого не требует.
- Не добавляй новые зависимости, если они не нужны для выполнения задачи.
- Не редактируй сгенерированные файлы, результаты сборки, lock-файлы и файлы окружения, если это явно не требуется.
- Никогда не добавляй в код секреты, токены, пароли, API-ключи и приватные значения конфигурации.
- Держи изменения минимальными и строго связанными с запросом пользователя.

## Правила для сайта

- Сохраняй адаптивность страниц для мобильных устройств и компьютеров.
- Соблюдай базовую доступность: семантический HTML, alt-текст для значимых изображений, элементы управления, доступные с клавиатуры, и читаемый контраст.
- Не ломай базовое SEO: заголовки, описания, структуру заголовков, canonical URL, sitemap, robots rules и структурированные метаданные, если они уже есть.
- Предпочитай переиспользуемые компоненты вместо дублирования разметки или логики.
- Сохраняй изменения интерфейса в рамках существующей дизайн-системы.

## Качество кода

- Следуй стилю, который уже используется в репозитории.
- Предпочитай простой и понятный код вместо слишком умных абстракций.
- Удаляй неиспользуемый код, импорты и мёртвые ветки, которые появились из-за изменений.
- При изменении пользовательской функциональности учитывай состояния загрузки, пустого результата и ошибки.

## Проверки

Перед завершением задачи запусти подходящие проверки, доступные в проекте, например:

- устанавливай зависимости только при необходимости;
- запусти линтинг, если он настроен;
- запусти проверку типов, если она настроена;
- запусти тесты, если они настроены;
- запусти команду сборки, если изменение может повлиять на production-поведение.

Если какую-то проверку невозможно выполнить, объясни почему.

## Формат ответа

После завершения задачи кратко опиши:

- что было изменено;
- какие файлы были затронуты;
- какие проверки были запущены;
- что пользователю стоит проверить вручную.

## Чистота репозитория

Не добавляйте в Git генерируемые, временные и локальные файлы:

- кэши и результаты сборки;
- логи;
- настройки IDE;
- файлы операционной системы;
- локальные данные и загрузки;
- служебные файлы тестовых инструментов;
- файлы окружения с секретами.

## Рабочая область

Этот проект является сайтом внутри экосистемы.

Работать только внутри `website-blog/`.
Соседние проекты не читать и не изменять без прямого запроса пользователя.

Git-репозиторий находится уровнем выше, в `C:\zz-ecosystem`.
Для проверки статуса, коммитов и push использовать корень `C:\zz-ecosystem`.
